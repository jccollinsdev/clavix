## Key Findings

1. **AI/LLM data flow is undisclosed.** Portfolio context (tickers, share counts, cost basis) is sent to MiniMax for risk scoring and digest generation. The current Privacy Policy does not mention this.
2. **Analytics claim is false.** The Policy claims "usage data" collection but no SDK exists.
3. **Brokerage claims are premature.** The Policy and Terms describe brokerage connections as a live feature. Brokerage is disabled via FeatureFlags.brokerageEnabled=false.
4. **Push notification token handling is missing** from both legal docs.
5. **Device token, name, birth_year, and all preferences** are not listed in the Privacy Policy.
6. **No privacy manifest (PrivacyInfo.xcprivacy)** exists in the iOS app.
7. **Sentry SDK** is in the backend code but disabled by default (no DSN configured). If configured, it would capture error traces with request context.
8. **Shared ticker data (`shared_ticker_events`, `prices`)** is publicly readable (no auth required). This should be disclosed as public data.
