# Session Log — 2026-06-16

## What We Did

### 1. Grade Flicker Fix (3 separate bugs)

**Problem:** Ticker grades were flipping letter grades day-to-day across all stocks, not just AAPL.

**Bug 1 — missing `grade` column in snapshot SELECT**
`backend/app/services/ticker_cache_service.py` — the previous snapshot query was:
```python
.select("composite_score,safety_score,snapshot_date")
```
Missing `grade` meant `prev_grade` was always `None`, so `apply_grade_hysteresis()` never fired.
Fixed to:
```python
.select("composite_score,safety_score,snapshot_date,grade")
```

**Bug 2 — limited-data tickers bypassed EMA smoothing**
EMA smoothing ran on `total_score`, then limited-data recompute (for tickers with <7 days news) overwrote with an unsmoothed value. Fixed by re-applying `smooth_score_with_history()` after the limited-data recompute.

**Bug 3 — hysteresis buffer too small**
`backend/app/pipeline/analysis_utils.py` — raised `_HYSTERESIS_BUFFER` from `2.0` to `3.0` (prod data showed average flip delta of 4.8 points).

---

### 2. Stale Tickers Fixed (SPY, VOO, HIMS, POET, all ETFs)

**Problem:** `list_active_sp500_tickers()` filtered to `index_membership = 'SP500'` only, leaving ETF/CORE_ETF/USER_SHARED tickers (SPY, VOO, etc.) out of the daily recompute entirely.

**Fix in `backend/app/services/ticker_cache_service.py`:** Now queries all `is_active=True` tickers, ordered SP500 first, then ETF/CORE_ETF, then USER_SHARED.

---

### 3. Push Notification Prompt Fix

**Problem:** `ClavisApp.swift` called `registerForRemoteNotifications()` directly without ever calling `requestPermission()` first. iOS silently ignores registration without prior authorization — users never saw the permission dialog.

**Fix in `ios/Clavis/App/ClavisApp.swift`:**
```swift
// Before:
PushNotificationManager.shared.registerForRemoteNotifications()
// After:
Task { await PushNotificationManager.shared.requestPermission() }
```

---

### 4. Apple Sign-In Error 1000 Fix

**Problem:** `com.apple.AuthenticationServices.AuthorizationError error 1000` on physical device when tapping "Sign in with Apple."

**Root cause:** `ASAuthorizationController` was a local variable inside the `withCheckedThrowingContinuation` closure in `AppleSignInCoordinator.signIn()`. On real devices it got deallocated before the Apple auth sheet finished presenting.

**Fix in `ios/Clavis/Services/SupabaseAuthService.swift`:**
- Added `private var controller: ASAuthorizationController?` property to `AppleSignInCoordinator`
- Assigned `self.controller = controller` before `performRequests()`
- Cleared `self.controller = nil` in both delegate callbacks (success and error)
- Also fixed `presentationAnchor(for:)` to use `MainActor.assumeIsolated { }` instead of calling `UIApplication.shared` from a `nonisolated` context (eliminated 4 compiler warnings)

---

### 5. Archive Validation Errors Fixed (iPad requirements)

**Problem:** App Store Connect upload failed with two validation errors:
- Missing 152x152 iPad icon
- Missing landscape orientations for iPad multitasking

**Root cause:** Xcode project had `TARGETED_DEVICE_FAMILY = "1,2"` (Universal: iPhone + iPad) in Release and other configurations, despite Clavix being iPhone-only.

**Fix in `ios/Clavis.xcodeproj/project.pbxproj`:** Changed all `"1,2"` to `1` (iPhone only) across all configurations.

```bash
sed -i '' 's/TARGETED_DEVICE_FAMILY = "1,2";/TARGETED_DEVICE_FAMILY = 1;/g' ios/Clavis.xcodeproj/project.pbxproj
```
